Mendeteksi Outlier Menggunakan K-Nearest Neighbors (KNN) dalam Analisis Data#

1. Pengenalan Outlier dalam Analisis Data#

Outlier adalah data yang memiliki nilai yang sangat berbeda dibandingkan dengan sebagian besar data lainnya dalam suatu dataset. Outlier dapat muncul karena berbagai faktor, seperti kesalahan pengukuran, kesalahan saat memasukkan data, atau fenomena yang jarang terjadi. Dalam tahap pemahaman data, penting untuk mengidentifikasi outlier karena keberadaannya dapat berdampak pada hasil analisis, pemodelan, serta pengambilan keputusan. Jika tidak terdeteksi, outlier dapat menyebabkan model analisis statistik atau machine learning menjadi kurang akurat, bias, atau bahkan gagal menghasilkan prediksi yang valid.

2. Pendekatan K-Nearest Neighbors (KNN) dalam Mendeteksi Outlier#

Salah satu metode yang dapat digunakan untuk mendeteksi outlier adalah K-Nearest Neighbors (KNN). Algoritma ini bekerja berdasarkan prinsip bahwa sebagian besar data normal akan berada dalam kelompok yang padat, sementara outlier cenderung berada jauh dari kelompok utama. Berikut adalah cara kerja dasar KNN dalam mendeteksi outlier:

  • Mekanisme Kerja KNN:

KNN menghitung jarak antara satu titik data dengan titik-titik data di sekitarnya. Jika suatu titik memiliki jarak yang cukup jauh dari sebagian besar tetangganya, maka titik tersebut bisa dianggap sebagai outlier. Secara matematis, KNN mengukur jarak menggunakan berbagai metode seperti Euclidean, Manhattan, atau Minkowski untuk menentukan k-tetangga terdekat. Jika rata-rata jarak ke k-tetangga lebih besar dari nilai ambang tertentu, maka titik tersebut dikategorikan sebagai outlier.

Kapan Menggunakan KNN untuk Deteksi Outlier?#

KNN merupakan metode yang berguna dalam mendeteksi outlier dalam beberapa kondisi berikut

Data dengan Dimensi Tinggi: KNN mampu bekerja pada dataset dengan banyak fitur, meskipun kinerjanya dapat menurun jika jumlah dimensi terlalu besar akibat efek “curse of dimensionality”.

Data Tanpa Distribusi Tertentu: Algoritma ini tidak memerlukan asumsi tentang distribusi data, sehingga cocok digunakan pada dataset yang tidak mengikuti distribusi normal atau tertentu.

Dataset Berukuran Sedang: KNN lebih optimal digunakan pada dataset yang tidak terlalu besar karena kompleksitas komputasi meningkat seiring bertambahnya jumlah data. Jika dataset terlalu besar, metode lain seperti Local Outlier Factor (LOF) atau Isolation Forest bisa menjadi pilihan yang lebih efisien.

Kesimpulan#

K-Nearest Neighbors (KNN) adalah metode yang cukup efektif dalam mengidentifikasi outlier, terutama pada tahap eksplorasi data. Metode ini memanfaatkan perhitungan jarak untuk menemukan titik data yang memiliki perbedaan signifikan dibandingkan sebagian besar data lainnya. Meskipun memiliki kelebihan seperti fleksibilitas dan kemudahan implementasi, KNN juga memiliki beberapa kekurangan, terutama dalam hal efisiensi komputasi serta pemilihan parameter yang tepat. Oleh karena itu, penting untuk memahami karakteristik dataset sebelum memilih KNN sebagai metode deteksi outlier.

Dalam praktiknya, KNN sering dikombinasikan dengan teknik lain, seperti visualisasi data (misalnya scatter plot atau box plot), untuk memverifikasi hasil deteksi outlier dan memastikan kualitas data yang lebih baik sebelum melanjutkan ke tahap pemodelan.

# Ambil data dari kedua database
df_postgresql = get_pg_data()
df_mysql = get_mysql_data()

# Gabungkan berdasarkan kolom 'id' dan 'Class'
df_merged = pd.merge(df_mysql, df_postgresql, on=["id", "class"], how="inner")

# Ambil data fitur numerik
feature_columns = ["petal_length", "petal_width", "sepal_length", "sepal_width"]
data_values = df_merged[feature_columns].values

# Gunakan KNN untuk mendeteksi outlier
K = 5  # Jumlah tetangga terdekat
nbrs = NearestNeighbors(n_neighbors=K+1, metric='euclidean').fit(data_values)
distances, indices = nbrs.kneighbors(data_values)

# Ambil rata-rata jarak ke K tetangga terdekat (tidak termasuk jarak ke dirinya sendiri)
df_merged["knn_distance"] = np.mean(distances[:, 1:], axis=1)

# Tentukan threshold outlier berdasarkan persentil ke-95
threshold = np.percentile(df_merged["knn_distance"], 95)
df_merged["outlier"] = df_merged["knn_distance"] > threshold

# Cetak hasil data dengan outlier
print(df_merged.to_string(index=False))

Local Outlier Factor (LOF)#

Local Outlier Factor (LOF) adalah algoritma yang digunakan untuk mendeteksi anomali atau data yang menyimpang dalam suatu dataset. Algoritma ini bekerja dengan mengukur kepadatan lokal di sekitar suatu titik data dan membandingkannya dengan kepadatan tetangga-tetangganya. Jika suatu titik memiliki kepadatan yang jauh lebih rendah dibandingkan dengan titik-titik di sekitarnya, maka titik tersebut lebih mungkin dianggap sebagai outlier.

Proses LOF dimulai dengan menentukan jumlah tetangga terdekat (K), menghitung jarak ke tetangga-tetangga tersebut, dan kemudian menentukan faktor LOF berdasarkan rasio kepadatan lokal. Nilai LOF mendekati 1 menunjukkan titik yang normal, sementara nilai yang jauh lebih besar dari 1 mengindikasikan adanya potensi outlier. LOF sangat berguna dalam mendeteksi anomali pada data dengan kepadatan tidak merata, seperti transaksi keuangan yang mencurigakan atau pola serangan siber yang jarang terjadi.

Meskipun memiliki akurasi yang baik, LOF dapat menjadi lambat pada dataset berukuran besar dan cukup sensitif terhadap pemilihan parameter K. Namun, fleksibilitas dan kemampuannya dalam mendeteksi outlier secara kontekstual menjadikannya alat yang sangat berguna dalam analisis data.

Tahapan dalam LOF#

  1. Menentukan Parameter

  • k: Jumlah tetangga terdekat yang akan dipertimbangkan (biasanya disebut sebagai min_samples).

  • Metrik Jarak: Metode yang digunakan untuk mengukur jarak antara titik-titik data (misalnya, Euclidean distance).

  1. Menghitung Jarak Antar Titik

  • Menghitung jarak antara setiap titik dalam dataset dengan seluruh titik lainnya menggunakan metrik yang telah ditentukan.

  1. Menentukan Tetangga Terdekat (k-Nearest Neighbors)

  • Untuk setiap titik data, cari k tetangga terdekat berdasarkan hasil perhitungan jarak sebelumnya.

  1. Menghitung Reachability Distance (Jarak Jangkauan)

  • Untuk setiap titik, hitung jarak jangkauan terhadap tetangga-tetangga terdekatnya.

  • Jarak jangkauan antara titik A dan B didefinisikan sebagai: $\(reach-dist k(A,B)=max(distance(A,B),k-distance(B)) \)$

  1. Menghitung Local Reachability Density (LRD)

  • Hitung Local Reachability Density (LRD) untuk setiap titik, yang merupakan kebalikan dari rata-rata jarak jangkauan ke tetangga terdekatnya:

\[\text{LRD}(A) = \frac{1}{\frac{1}{k} \sum_{B \in N_k(A)} d_{\text{reach}}(A, B)} \]

  1. Menghitung Local Outlier Factor (LOF)

  • Nilai LOF dihitung sebagai rata-rata rasio LRD dari tetangga terdekat terhadap LRD titik tersebut: $\(\text{LOF}_k(A) = \frac{\sum_{B \in N_k(A)} \frac{\text{LRD}_k(B)}{\text{LRD}_k(A)}}{|N_k(A)|} \)$

  • Jika nilai LOF jauh lebih besar dari 1, maka titik tersebut kemungkinan besar merupakan outlier, karena kepadatan lokalnya lebih rendah dibandingkan dengan tetangga-tetangganya.

  1. Mengidentifikasi Outlier

  • Titik-titik dengan nilai LOF tinggi (biasanya >1) dianggap sebagai outlier, sedangkan titik dengan nilai LOF mendekati 1 merupakan data yang normal.

Contoh Perhitungan LOF Secara Manual#

Misalkan kita memiliki 4 titik data dalam ruang dua dimensi: $\(A(1,1),B(1,2),C(2,2),D(8,8) \)$

  1. Menghitung Jarak Antar Titik (Euclidean Distance)

\[\begin{split}\begin{gathered}\text { distance }(A, B)=\sqrt{(1-1)^2+(1-2)^2}=1 \\ \text { distance }(A, C)=\sqrt{(1-2)^2+(1-2)^2}=\sqrt{2} \approx 1.41 \\ \text { distance }(A, D)=\sqrt{(1-8)^2+(1-8)^2}=9.89 \\ \text { distance }(B, C)=\sqrt{(1-2)^2+(2-2)^2}=1 \\ \text { distance }(B, D)=\sqrt{(1-8)^2+(2-8)^2}=9.22 \\ \text { distance }(C, D)=\sqrt{(2-8)^2+(2-8)^2}=8.49\end{gathered} \end{split}\]

  1. Menentukan K Tetangga Terdekat

Untuk titik D, dua tetangga terdekat adalah B dan C.

  1. Hitung Reachability Distance (K=2) \(\begin{aligned} & \operatorname{reach-dist}(D, B)=\max (\mathrm{k}-\operatorname{dist}(B), \operatorname{distance}(D, B))=\max (1,9.22)=9.22 \\ & \operatorname{reach}-\operatorname{dist}(D, C)=\max (\mathrm{k} \text {-dist }(C), \operatorname{distance}(D, C))=\max (1,8.49)=8.49\end{aligned}\)

  2. Hitung Local Reachability Density (LRD)

\[\begin{split} \begin{gathered} \operatorname{LRD}(D)=\frac{1}{\frac{9.22+8.49}{2}}=\frac{1}{8.855} \approx 0.113 \\ \operatorname{LRD}(B)=\frac{1}{1}=1 \\ \operatorname{LRD}(C)=\frac{1}{1}=1 \end{gathered} \end{split}\]

  1. Hitung Local Outier Factor (LOF)

\[ \operatorname{LOF}(D)=\frac{1}{2}\left(\frac{1}{0.113}+\frac{1}{0.113}\right)=\frac{1}{2}(8.85+8.85)=8.85 \]

  1. Interpretasi

LOF \((\mathrm{D}) \approx 8.85 \rightarrow\) Outlier (karena jauh lebih besar dari 1)

LOF \((A, B, C) \approx 1 \rightarrow\) Normal Points

import pandas as pd

df_mysql = get_mysql_data()

# Gabungkan berdasarkan kolom 'id' dan 'Class'
df_merged = pd.merge(df_mysql, df_postgresql, on=["id", "class"], how="inner")

# Ambil data fitur numerik
feature_columns = ["petal_length", "petal_width", "sepal_length", "sepal_width"]
data_values = df_merged[feature_columns].values

# Gunakan KNN untuk mendeteksi outlier
K = 5  # Jumlah tetangga terdekat
nbrs = NearestNeighbors(n_neighbors=K+1, metric='euclidean').fit(data_values)
distances, indices = nbrs.kneighbors(data_values)

# Ambil rata-rata jarak ke K tetangga terdekat (tidak termasuk jarak ke dirinya sendiri)
df_merged["knn_distance"] = np.mean(distances[:, 1:], axis=1)

# Tentukan threshold outlier berdasarkan persentil ke-95
threshold = np.percentile(df_merged["knn_distance"], 95)
df_merged["outlier"] = df_merged["knn_distance"] > threshold

import numpy as np
import pandas as pd
from sklearn.datasets import load_iris
from sklearn.neighbors import LocalOutlierFactor

# Load dataset
iris = load_iris()
df = pd.DataFrame(iris.data, columns=['sepal_length', 'sepal_width', 'petal_length', 'petal_width'])

# Tambahkan kolom id
df["id"] = df.index

# Tambahkan kolom kelas (target)
df["species"] = iris.target
species_mapping = {0: "setosa", 1: "versicolor", 2: "virginica"}
df["species"] = df["species"].map(species_mapping)

# Ambil hanya data dengan spesies 'versicolor'
df_versicolor = df[df["species"] == "versicolor"].copy()

# Ambil fitur numerik untuk LOF
X = df_versicolor[['petal_length', 'petal_width', 'sepal_length', 'sepal_width']]

# Inisialisasi Local Outlier Factor (LOF)
clf = LocalOutlierFactor(n_neighbors=20, contamination=0.1)
df_versicolor["outlier"] = clf.fit_predict(X)  # -1 = outlier, 1 = inlier

# Hitung rata-rata petal_length tanpa outlier
mean_petal_length = df_versicolor[df_versicolor["outlier"] == 1]["petal_length"].mean()

# Ganti petal_length outlier dengan rata-rata yang sudah dihitung
df_versicolor.loc[df_versicolor["outlier"] == -1, "petal_length"] = mean_petal_length

# Hapus kolom 'outlier' agar data bersih
df_versicolor.drop(columns=["outlier"], inplace=True)

# Cetak hasil akhir
print(df_versicolor[["id", "sepal_length", "sepal_width", "petal_length", "petal_width", "species"]])

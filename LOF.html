Local Outlier Factor (LOF)#

Local Outlier Factor (LOF) adalah algoritma yang digunakan untuk mendeteksi anomali atau data yang menyimpang dalam suatu dataset. Algoritma ini bekerja dengan mengukur kepadatan lokal di sekitar suatu titik data dan membandingkannya dengan kepadatan tetangga-tetangganya. Jika suatu titik memiliki kepadatan yang jauh lebih rendah dibandingkan dengan titik-titik di sekitarnya, maka titik tersebut lebih mungkin dianggap sebagai outlier.

Proses LOF dimulai dengan menentukan jumlah tetangga terdekat (K), menghitung jarak ke tetangga-tetangga tersebut, dan kemudian menentukan faktor LOF berdasarkan rasio kepadatan lokal. Nilai LOF mendekati 1 menunjukkan titik yang normal, sementara nilai yang jauh lebih besar dari 1 mengindikasikan adanya potensi outlier. LOF sangat berguna dalam mendeteksi anomali pada data dengan kepadatan tidak merata, seperti transaksi keuangan yang mencurigakan atau pola serangan siber yang jarang terjadi.

Meskipun memiliki akurasi yang baik, LOF dapat menjadi lambat pada dataset berukuran besar dan cukup sensitif terhadap pemilihan parameter K. Namun, fleksibilitas dan kemampuannya dalam mendeteksi outlier secara kontekstual menjadikannya alat yang sangat berguna dalam analisis data.

Tahapan dalam LOF#

  1. Menentukan Parameter

  • k: Jumlah tetangga terdekat yang akan dipertimbangkan (biasanya disebut sebagai min_samples).

  • Metrik Jarak: Metode yang digunakan untuk mengukur jarak antara titik-titik data (misalnya, Euclidean distance).

  1. Menghitung Jarak Antar Titik

  • Menghitung jarak antara setiap titik dalam dataset dengan seluruh titik lainnya menggunakan metrik yang telah ditentukan.

  1. Menentukan Tetangga Terdekat (k-Nearest Neighbors)

  • Untuk setiap titik data, cari k tetangga terdekat berdasarkan hasil perhitungan jarak sebelumnya.

  1. Menghitung Reachability Distance (Jarak Jangkauan)

  • Untuk setiap titik, hitung jarak jangkauan terhadap tetangga-tetangga terdekatnya.

  • Jarak jangkauan antara titik A dan B didefinisikan sebagai: \(reach-dist k(A,B)=max(distance(A,B),k-distance(B)) \)

  1. Menghitung Local Reachability Density (LRD)

  • Hitung Local Reachability Density (LRD) untuk setiap titik, yang merupakan kebalikan dari rata-rata jarak jangkauan ke tetangga terdekatnya:

\(\text{LRD}(A) = \frac{1}{\frac{1}{k} \sum_{B \in N_k(A)} d_{\text{reach}}(A, B)} \) 6. Menghitung Local Outlier Factor (LOF)

  • Nilai LOF dihitung sebagai rata-rata rasio LRD dari tetangga terdekat terhadap LRD titik tersebut: \(\text{LOF}_k(A) = \frac{\sum_{B \in N_k(A)} \frac{\text{LRD}_k(B)}{\text{LRD}_k(A)}}{|N_k(A)|} \)

  • Jika nilai LOF jauh lebih besar dari 1, maka titik tersebut kemungkinan besar merupakan outlier, karena kepadatan lokalnya lebih rendah dibandingkan dengan tetangga-tetangganya.

  1. Mengidentifikasi Outlier

  • Titik-titik dengan nilai LOF tinggi (biasanya >1) dianggap sebagai outlier, sedangkan titik dengan nilai LOF mendekati 1 merupakan data yang normal.

Contoh Perhitungan LOF Secara Manual#

Misalkan kita memiliki 4 titik data dalam ruang dua dimensi: \(A(1,1),B(1,2),C(2,2),D(8,8) \)

  1. Menghitung Jarak Antar Titik (Euclidean Distance)

\(\begin{gathered}\text { distance }(A, B)=\sqrt{(1-1)^2+(1-2)^2}=1 \\ \text { distance }(A, C)=\sqrt{(1-2)^2+(1-2)^2}=\sqrt{2} \approx 1.41 \\ \text { distance }(A, D)=\sqrt{(1-8)^2+(1-8)^2}=9.89 \\ \text { distance }(B, C)=\sqrt{(1-2)^2+(2-2)^2}=1 \\ \text { distance }(B, D)=\sqrt{(1-8)^2+(2-8)^2}=9.22 \\ \text { distance }(C, D)=\sqrt{(2-8)^2+(2-8)^2}=8.49\end{gathered} \)

  1. Menentukan K Tetangga Terdekat

Untuk titik D, dua tetangga terdekat adalah B dan C.

  1. Hitung Reachability Distance (K=2) \(\begin{aligned} & \operatorname{reach-dist}(D, B)=\max (\mathrm{k}-\operatorname{dist}(B), \operatorname{distance}(D, B))=\max (1,9.22)=9.22 \\ & \operatorname{reach}-\operatorname{dist}(D, C)=\max (\mathrm{k} \text {-dist }(C), \operatorname{distance}(D, C))=\max (1,8.49)=8.49\end{aligned}\)

  2. Hitung Local Reachability Density (LRD)

\( \begin{gathered} \operatorname{LRD}(D)=\frac{1}{\frac{9.22+8.49}{2}}=\frac{1}{8.855} \approx 0.113 \\ \operatorname{LRD}(B)=\frac{1}{1}=1 \\ \operatorname{LRD}(C)=\frac{1}{1}=1 \end{gathered} \)

  1. Hitung Local Outier Factor (LOF)

\( \operatorname{LOF}(D)=\frac{1}{2}\left(\frac{1}{0.113}+\frac{1}{0.113}\right)=\frac{1}{2}(8.85+8.85)=8.85 \)

  1. Interpretasi

LOF \((\mathrm{D}) \approx 8.85 \rightarrow\) Outlier (karena jauh lebih besar dari 1)

LOF \((A, B, C) \approx 1 \rightarrow\) Normal Points

Implementasi menggunakan Sklearn#

import pandas as pd
import numpy as np
import seaborn as sns
import matplotlib.pyplot as plt
from sklearn.neighbors import NearestNeighbors

# Ambil data dari kedua database
df_postgresql = get_pg_data()
df_mysql = get_mysql_data()

# Gabungkan berdasarkan kolom 'id' dan 'Class'
df_merged = pd.merge(df_mysql, df_postgresql, on=["id", "class"], how="inner")

# Ambil data fitur numerik
feature_columns = ["petal_length", "petal_width", "sepal_length", "sepal_width"]
data_values = df_merged[feature_columns].values

# Gunakan KNN untuk mendeteksi outlier
K = 5  # Jumlah tetangga terdekat
nbrs = NearestNeighbors(n_neighbors=K+1, metric='euclidean').fit(data_values)
distances, indices = nbrs.kneighbors(data_values)

# Ambil rata-rata jarak ke K tetangga terdekat (tidak termasuk jarak ke dirinya sendiri)
df_merged["knn_distance"] = np.mean(distances[:, 1:], axis=1)

# Tentukan threshold outlier berdasarkan persentil ke-95
threshold = np.percentile(df_merged["knn_distance"], 95)
df_merged["outlier"] = df_merged["knn_distance"] > threshold

# Cetak 10 data dari Sepal
print("10 Data Sepal:")
print(df_merged[["id", "class", "sepal_length", "sepal_width", "outlier"]].head(10).to_string(index=False))

# Cetak 10 data dari Petal
print("\n10 Data Petal:")
print(df_merged[["id", "class", "petal_length", "petal_width", "outlier"]].head(10).to_string(index=False))

# Visualisasi scatter plot Sepal
plt.figure(figsize=(8, 6))
sns.scatterplot(
    x=df_merged["sepal_length"], y=df_merged["sepal_width"],
    hue=df_merged["outlier"], palette={False: "blue", True: "red"}
)
plt.title("Outlier Detection with KNN (Sepal)")
plt.show()

# Visualisasi scatter plot Petal
plt.figure(figsize=(8, 6))
sns.scatterplot(
    x=df_merged["petal_length"], y=df_merged["petal_width"],
    hue=df_merged["outlier"], palette={False: "blue", True: "red"}
)
plt.title("Outlier Detection with KNN (Petal)")
plt.show()

Code untuk mendeteksi outlier dalam dataset Iris.#

import numpy as np
import pandas as pd
from sklearn.datasets import load_iris
from sklearn.neighbors import LocalOutlierFactor

# Load dataset
iris = load_iris()
df = pd.DataFrame(iris.data, columns=iris.feature_names)

# Ganti nama kolom agar sesuai dengan permintaan
df.columns = ['sepal_length', 'sepal_width', 'petal_length', 'petal_width']

# Pastikan df_merged didefinisikan (jika memang perlu merging, lakukan di sini)
df_merged = df  # Jika tidak ada proses merge, gunakan df langsung

# Ambil fitur yang diminta
x = df_merged[['petal_length', 'petal_width', 'sepal_length', 'sepal_width']]

# Inisialisasi model Local Outlier Factor
clf = LocalOutlierFactor(n_neighbors=20)

# Prediksi anomali/outlier
y_pred = clf.fit_predict(x)

# Cetak hasil prediksi
print(y_pred)
